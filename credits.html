Security concerns

The two most common vulnerabilities that can affect a website such as this are SQL injection and cross-site-scripting (XSS). An SQL injection occurs when an attacker is able to manipulate the input of a websites SQL query which in this case is the inputs on the find out more page.

Some potential security concerns that can arise from SQL injections are:

  • Unauthorised data access. Allowing an attacker to extract sensitive data from the database, however the database is shown in full on the view requests page but under usual circumstances this would not be the case.
  • Data modification or deletion. An attacker can modify or delete data from the database causing data loss or tampering.
  • Denial of service (DoS). An attacker executing malicious queries may cause the websites database to become overwhelmed.

To combat SQL injections I have used the real_escape_string() function in my submit.php file when defining the variables. Another method I could include to combat this attack is the principle of least privilege, which ensures the user has limited permissions which minimises the potential damage of a successful attack.

Some potential security concerns that can arise from XSS are:

  • Session hacking. Allowing attackers to steal user session cookies therefore allowing them to impersonate the user and perform actions on their behalf.
  • Defacement. Attackers can modify the appearance of the website.
  • Information theft. An attacker can steal sensitive user data such as login details or personal data.

To combat cross-site-scripting (XSS) I have used the htmlspecialchars() function which converts special predefined characters to html entries. This is know as output encoding. Another method I could include to combat this attack is a Content Security Policy (CSP), which specifies which sources of content are considered trustworthy therefore reducing the impact of a XSS attack.

The final security concern I have considered yet have no means to prevent is the lack of a Secure Socket Layer (SSL) as the hosting used on www.numyspace.co.uk does not have this feature. This leaves the website vulnerable to packet sniffing.
